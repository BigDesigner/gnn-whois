# AI Workflow Pipeline

## Workflow
1. **Research:** Consult official documentation and security best practices for the specific task.
2. **Plan:** Research and create an implementation plan based on official standards.
3. **Execute:** Implement changes in atomic steps using secure-by-default functions.
4. **Verify:** Run Pass/Fail checks (Syntax, Logic, Standards). **CRITICAL: If FAIL, return to step 3. DO NOT COMMIT.**
5. **Commit & Log:** Once PASS is confirmed, commit the changes and update the Memory Bank/worklog with source references.
6. **Clean Up:** Ensure a clean worktree before concluding the session.

## Automation
- GitHub Actions for linting and deployment.
- Pre-commit hooks for basic code quality.
